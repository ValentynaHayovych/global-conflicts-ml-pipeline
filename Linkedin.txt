I'm grateful to share a project that evolved over several weeks of experimentation.

Built an end-to-end ML pipeline analyzing synthetic global conflicts data with:
- rule-based data validation
- anomaly detection
- clustering
- interactive geographic visualization
- prediction models
- comparative ML analysis

The pipeline has multiple stages:
data ingestion → validation → anomaly detection → prediction models → visualization → final evaluation

Implemented:
- Isolation Forest
- Local Outlier Factor (LOF)
- DBSCAN
- Logistic Regression
- Random Forest
- Folium interactive mapping

Comparing rule-based validation against ML-based anomaly detection gave an interesting result:
Rule validation aggressively rejected records using country, year, population and coordinates constraints, while ML models still detected statistically "valid-looking" patterns within noisy synthetic data.

That led to one useful realization:
A dataset can look structurally consistent to ML while still failing real-world factual validation.

Curious observations - with a few minor absurdities:
- only 1.5% of global conflicts data passed fact checks
- land filtering retained just 36% of records
- DBSCAN expectedly is the most interpretable anomaly detection for geographic noise
- prediction models performed near random chance (~50%), exposing weak feature-target relationships in the synthetic dataset

On balance, the synthetic features carried remarkably little predictive value, leaving data quality engineering to overshadow the modeling almost entirely.
